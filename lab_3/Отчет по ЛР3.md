## Алгоритм работы программы

### 1. Входные данные
На вход подаются:
- блок данных M фиксированной длины
- секретный ключ K

### 2. Подготовка
1. Блок M делится на две равные части:
   L0, R0
2. Из ключа K генерируются раундовые ключи:
   K1, K2, ..., Kn

### 3. Основной цикл (n раундов)

Для каждого раунда i = 1..n:

    Li = R(i-1)
    Ri = L(i-1) XOR F(R(i-1), Ki)

где:
- F — раундовая функция
- XOR — операция исключающего ИЛИ

---

### 4. Результат шифрования
После выполнения всех раундов:

    C = (Ln, Rn)

(иногда выполняется дополнительная перестановка половин)


### 5. Алгоритм расшифрования

1. Вход:
    $C = (L_n, R_n)$

2. Используются те же операции, но ключи в обратном порядке:
    $K_n, K_{n-1}, ..., K_1$

3. На каждом шаге:
	$R_{i-1} = L_{i}$
    $L_{i-1} = R_{i} \oplus F(L_{i}, K_{i})$

## Контрольные вопросы

>[!Question] Какова структура классической сети Фейстеля?
>>Классическая сеть Фейстеля имеет следующую структуру. Входной блок делится на несколько равной длины подблоков, называемых ветвями сети. В классической схеме их две

>[!Question] Что называется раундом в сети Фейстеля?
>>Действие, состоящее из однократного вычисления образующей функции и последующего наложения (сложения по модулю 2) ее результата на другую ветвь с обменом их местами, называется циклом или раундом

>[!Question] Какими свойствами обладает сеть Фейстеля?
>> 1. Обратимость
>> 2. Симметричность шифрования и дешифрования
>> 3. Конфузия - скрывает связь между ключом и шифротекстом
>> 4. Диффузия - один бит изменяет множество бит 

>[!Question] Каким образом используется материал ключа при шифровании?
>>Ключ искажает данные и делает их непредсказуемыми из функции, в которую передают ключ и часть данных
